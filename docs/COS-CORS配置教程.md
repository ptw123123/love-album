# 腾讯云 COS 存储桶 CORS 配置教程

为了让相册的「浏览器直传」生效（上传更快、不经过服务器），需要在腾讯云 COS 控制台为**当前使用的存储桶**配置跨域访问（CORS）。否则浏览器会拦截直传请求并报错。

---

## 一、登录并进入存储桶

1. 打开浏览器，访问腾讯云对象存储控制台：  
   **https://console.cloud.tencent.com/cos5**

2. 使用你的腾讯云账号登录（未注册需先注册）。

3. 在左侧导航栏点击 **「存储桶列表」**。

4. 在列表中找到 **love-album 使用的那个存储桶**（即你在 `.env.local` 里配置的 `TENCENT_COS_BUCKET` 对应的桶），**点击该存储桶名称**，进入该桶的详情页。

---

## 二、打开 CORS 配置页

1. 在存储桶详情页的左侧导航栏中，找到 **「安全管理」**。

2. 点击 **「安全管理」** 下的 **「跨域访问 CORS 设置」**。

3. 在 **「跨域访问 CORS 设置」** 区域，点击 **「添加规则」** 按钮。

---

## 三、填写 CORS 规则（推荐配置）

在弹窗或表单中按下面说明逐项填写。带 * 的为必填。

### 1. 来源 Origin（必填）

填写**允许发起跨域请求的网页来源**，即你的相册站点地址。每行一个，不要有多余空格或结尾斜杠。

**本地开发：**

```
http://localhost:3000
```

**已部署的线上站点（示例）：**

```
https://你的域名.com
```

若同时有多个环境，可以多行填写，例如：

```
http://localhost:3000
https://love-album.vercel.app
https://你的自定义域名.com
```

注意：

- 必须带协议：`http://` 或 `https://`。
- 非 80/443 端口时需写端口，如：`http://localhost:3000`。
- 不要用 `*` 作为来源（不利于安全，且部分场景会受限）。

---

### 2. 操作 Methods（必填）

勾选以下方法即可满足直传需求：

- **GET**
- **PUT**
- **POST**
- **HEAD**

（DELETE 可选，本相册直传不强制需要。）

---

### 3. Allow-Headers（允许的请求头）

建议直接填：

```
*
```

表示允许所有请求头，避免浏览器预检请求被拒绝。若无特殊安全要求，这样即可。

---

### 4. Expose-Headers（暴露给前端的响应头）

建议填写：

```
ETag
```

这样浏览器里可以拿到对象的 ETag，便于后续如需做缓存或校验。每行一个，若控制台是多行输入框可只写一行 `ETag`。

---

### 5. 超时 Max-Age（可选）

可填：

```
600
```

表示预检结果缓存 600 秒（10 分钟），减少重复 OPTIONS 请求。不填一般也可用。

---

### 6. 返回 Vary: Origin（可选）

若你的页面**同时存在**跨域和非跨域请求，建议勾选「返回 Vary: Origin」；若不确定，可先**不勾选**，直传失败再勾选试一次。

---

## 四、保存规则

1. 检查上述项是否与推荐一致（尤其是 **来源** 和 **Methods**）。

2. 点击 **「保存」** 或 **「确定」**。

3. 回到「跨域访问 CORS 设置」列表，应能看到刚添加的规则。

---

## 五、验证是否生效

1. 打开你的相册页面（本地或已部署的均可，需与你在「来源」里填的地址一致）。

2. 选择一张图片、选择分区，点击「上传到我们的相册」。

3. 若配置正确：
   - 上传会直接发往腾讯云 COS（不经过你的服务器），速度较快。
   - 上传成功后能看到新照片和链接。

4. 若仍失败：
   - 打开浏览器 **开发者工具（F12）→ Network（网络）**，看是否有请求被标红。
   - 若报错里出现 **CORS**、**Access-Control-Allow-Origin**、**跨域** 等字样，多半是 CORS 未生效，请检查：
     - 当前访问相册的地址是否和「来源 Origin」里填的**完全一致**（协议、域名、端口）。
     - 是否保存了规则并等待几秒后再试。
     - 是否改对了 **love-album 实际使用的那一个存储桶**。

---

## 六、配置小结（可复制对照）

| 配置项           | 推荐值 |
|------------------|--------|
| **来源 Origin**  | `http://localhost:3000` 以及你的线上域名（如 `https://你的域名.com`），每行一个 |
| **操作 Methods** | GET、PUT、POST、HEAD |
| **Allow-Headers** | `*` |
| **Expose-Headers** | `ETag` |
| **超时 Max-Age** | `600`（可选） |

---

## 七、常见问题

**Q：控制台找不到「安全管理」或「跨域访问 CORS 设置」？**  
A：确认当前是在 **对象存储 COS** 控制台（cos5），且进入的是**单个存储桶的详情页**，而不是「概览」或「存储桶列表」页。左侧菜单在桶详情下才会出现「安全管理」。

**Q：已经加了规则，为什么还是报 CORS 错误？**  
A：常见原因：  
1）访问相册的地址和 CORS 里填的「来源」不一致（例如写了 `https://xxx.com` 却用 `http://xxx.com` 打开）；  
2）写成了 `https://你的域名.com/`（末尾多了 `/`），应改为 `https://你的域名.com`；  
3）配置在了别的存储桶，而不是 `.env.local` 里 `TENCENT_COS_BUCKET` 对应的桶。

**Q：用 Vercel / 其他平台部署，来源填什么？**  
A：填你浏览器地址栏里打开的相册域名，例如：  
`https://你的项目.vercel.app` 或你绑定的自定义域名 `https://你的域名.com`。保证和实际访问的 URL 一致即可。

---

配置完成后，直传即可生效，上传会更快、更稳定。若仍有问题，可把浏览器控制台或 Network 里的报错信息贴出来便于排查。
