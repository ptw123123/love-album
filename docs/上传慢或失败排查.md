# 上传慢或失败 — 排查思路（不压缩图片）

几 MB 的图不算大，慢或失败多半是**网络路径或限制**的问题，不是体积本身。按下面顺序排查。

---

## 1. 先确认：直传有没有真正生效？

上传时文件应该**直接从浏览器发到腾讯云 COS**，而不是经过 Netlify。

**怎么查：**

1. 电脑打开相册页面，按 **F12** 打开开发者工具，切到 **Network（网络）**。
2. 选一张图、选分区，点「上传到我们的相册」。
3. 看列表里出现的请求：
   - 若有一个请求的地址是 **`https://love-album-xxxx.cos.xxx.myqcloud.com/`**（或类似 COS 域名），说明**直传已生效**，文件是浏览器 → COS。
   - 若只有一个 **`/api/upload`** 发到你自己的站点（如 `jjandptw.netlify.app`），说明直传没走成，走的是**经 Netlify 转发**。

**若直传没生效（只有 /api/upload）：**

- 多半是 **CORS** 或 **取签名失败**。
- 检查 COS 控制台该桶的 **跨域访问 CORS** 里，**来源** 是否包含：
  - `https://jjandptw.netlify.app`
  - 若本地测，还有 `http://localhost:3000`
- 看 Network 里 **`/api/upload-token`** 请求：状态码是否 200？若 4xx/5xx，看返回内容或 Console 报错。

直传生效后，上传一般会快很多，也不受 Netlify 的 6MB、10s 限制。

---

## 2. 直传已生效，但还是慢或超时

这时是 **浏览器 → COS** 这一段慢或断。

**常见原因：COS 桶地域和用户/网络离得太远。**

- 例如：桶在 **中国广州**，而你在海外或用的网络出口在海外，跨洲访问就会慢、易超时。
- 手机在 4G/5G 下更容易因延迟大、不稳定而失败。

**可以做的：**

- 若**用户主要在国内**：在 COS 控制台看当前桶的地域，选离用户近的（如华南选广州、华东选上海）。同一地域内会快很多。
- 若**用户在国内、桶也在国内**仍然慢：看是否走了代理/VPN，或运营商对直连 COS 有限制，可换网络或关代理再试。
- 若**用户多在海外**：考虑把桶建在**海外地域**（如新加坡、硅谷等），或使用腾讯云**全球加速**类能力，让上传走更近的节点。

---

## 3. 直传失败后走了「经服务器」回退

当前逻辑是：直传失败会回退到 **浏览器 → Netlify → COS**。

**Netlify 的限制：**

- 请求体大小有上限（约 **6MB**，不同计划可能不同），超过会返回 **413**。
- 函数执行有时间上限（约 **10 秒**），超时会 **504** 或类似超时。

所以：**大一点的照片、或网络稍慢，经 Netlify 上传就容易失败或“卡很久然后失败”。** 这不是图片该不该压缩的问题，而是这条路径本来就不适合传大图。

**正确方向：** 让**直传稳定生效**，避免走到这条回退路径。  
即：保证 CORS、签名都正常，且 COS 地域/网络对用户友好（见上面第 1、2 步）。

---

## 4. 手机上传特别容易失败

可能原因：

1. **直传在手机上没生效**（例如 CORS 在部分移动浏览器下表现不同、或请求被拦截），于是走了 Netlify 回退，受 6MB/10s 限制。
2. **直传到了 COS，但手机网络不稳定或延迟大**，到 COS 的请求超时或被中断。
3. **运营商或省流量模式** 限制了长连接、大请求。

**建议：**

- 在手机浏览器里打开相册，用「请求 desktop 版网站」或开发者工具（如 Chrome 远程调试）看上传请求是发到 **COS 域名** 还是 **Netlify**。若是 Netlify，重点先修直传（CORS/签名）。
- 确认 COS CORS 里来源包含 **`https://jjandptw.netlify.app`**（不要多写斜杠或路径）。
- 若直传在电脑正常、手机不正常，可尝试同一页在手机用 **Wi‑Fi** 上传，对比 4G/5G，排除运营商限制。

---

## 5. 小结：优先做这两件事

1. **确认直传生效**  
   用 F12 → Network 看上传时请求是否发到 **cos.myqcloud.com**，并确保 CORS、`/api/upload-token` 都正常。
2. **看 COS 地域是否合适**  
   用户主要在哪，桶就选哪里的地域；海外用户多则考虑海外区或全球加速。

这两点没问题后，几 MB 的图不压缩也通常能稳定、较快上传。若你愿意，可以贴一下：上传失败时 Network 里对应请求的地址、状态码和报错信息，我可以根据具体返回帮你判断是 CORS、超时还是 413。
